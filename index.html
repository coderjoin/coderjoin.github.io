Java Web 总结（Spring MVC为例）

转眼间，不知不觉学习Java Web也有几年，从最开始的原生的servlet到Struts2、Hibernate和Spring整合，到最后的Spring、Spring MVC、Mybites整合。开始的学习也不管理解，都是直接快速配置，然后启动就ok。写一篇Java web的总结也是梳理一下自己的理解。

需要的配置文件

  • web.xml
  • *-servlet.xml
    配置文件的内容就不详细列了，网上也很多

Java Web加载的流程

  • 启动WEB项目的时候,容器(如:Tomcat)会去读它的配置文件web.xml.读两个节点: 和
  • 容器创建一个ServletContext(上下文),这个WEB项目所有部分都将共享这个上下文
  • 容器将转化为键值对,并交给ServletContext
  • 创建监听实例
  • 加载filter
  • 加载servlet
  • 加载Spring，加载一些bean，如视图解析器，启用注解功能和包扫描

Web请求过程

  • 当Web容器接收到来自客户端的请求信息之后，会根据URL请求到Servlet队列中查找对应的Servlet对象，如果找到则直接使用，如果没有找到则加载对应的类，并创建对象。
  • 这里就有servlet的生命周期的问题，servlet的init方法只会被调用一次，不会反复创建对象，直到容器回收或者超时。
  • init方法被调用后，调用service方法，调用doGet或者doPost方法，如果重写service方法，需要用super.service();
  • 调用destory()销毁

Java 基础总结

一、基本数据类型
  • 8种基本数据类型：byte、short、int、long、float、double、boolean、char。
    二、String
  • 字符串的不可变性
  • intern方法在1.6和1.7的区别
  • StringBuffer（线程安全）和StringBuilder（线程不安全）的区别
    三、面向对象
  • 封装、继承、多态
  • is-a和has-a关系
  • override和overload关系
    四、异常
  • 继承Throwable接口，分为Exception和Error
  • Exception分为checked exception 和 unchecked exception
  • checked exception需要去catch，例如io exception
  • unchecked exception 例如 空指针、数组越界、参数不合法
    五、集合
  • Collections（静态帮助类）和Collection（接口）的区别
  • set和list是Collection的subinterfaces
  • 常用集合类的使用
  • ArrayList 和 LinkedList 和 Vector 的区别 、SynchronizedList 和 Vector 的区别、HashMap、HashTable、ConcurrentHashMap 区别
  • 特性和线程安全性问题
    六、IO
  • 基于字节、基于字符、基于网络（某种分类）
  • 装饰者模式
  • 同步、异步、阻塞、非阻塞、Linux 5 种 IO 模型
  • BIO、NIO 和 AIO 的区别、三种 IO 的用法与原理、netty
    七、Classloader
  • bootstarp classloader、extension classloader、application classloader
  • 双亲委托模型（从双亲检查加载确保安全）
    八、gc
  • 可达性分析（哪些可以作为起点）
  • 典型算法：mark-sweep、copying、mark-compact、generational collection
  • 7种垃圾收集器

以上就是自己顺着以前做过的Java language programming的ppt，脑海里回忆的Java 基础知识点，随便记录下来。高级部分，以后再说。。。。
