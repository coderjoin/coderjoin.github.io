Semaphore

  • 信号量实现多生产者多消费者
  • 可以限制并发执行任务的线程数量
  • semaphore.acquire()、semaphore.release()

Exchanger

  • 两个线程间传输数据，exchange()方法

CountDownLatch

  • 减法操作，不可重用，计数不为0，线程呈阻塞状态
  • 一个线程或者多个线程，等待另外一个线程或者多个线程完成某个事情之后才能继续进行。

CyclicBarrier

  • 加法操作
  • 多个线程之间相互等待，“多个线程之间”任何一个线程没有完成任务，则所有的线程都必须等待。

Phaser

  • 类Phaser提供了动态增减parties计数，通过若干个方法来控制多个线程之间同步运行的效果，还可以实现针对某一个线程取消同步运行的效果，而且支持在指定屏障处等待，在等待时还支持中断或非中断等功能。

Executor与ThreadPoolExecutor

  • Executor接口，Executors工厂类
  • newCachedThreadPool() 无界线程池，理论最大值Integer.MAX_VALUE
  • newFixedThreadPool(int ) 有界线程池
  • newSingleThreadExecutor 单一线程池
  • newScheduledThreadPool(int) 定长线程池
  • 类ThreadPoolExecutor最常用的构造方法：ThreadPoolExecutor(int corePoolSize, int maximumPoolSize, long keepAliveTime, TimeUnit unit, BlockingQueue workQueue)
    • corePoolSize：核心池大小
    • maximunPoolSize：池中允许的最大线程数
    • keepAliveTime：当线程数量大于corePoolSize值时，在没有超过指定的时间内是不从线程池中将空闲线程删除的，如果超过此时间单位，则删除。
    • unit ：keepAliveTime参数的时间单位
    • workQueue：执行前用与保持任务的队列

      线程池ThreadPoolExecutor的拒绝策略

  • AbortPolicy：当任务添加到线程池中被拒绝时，抛出RejectedExecutionException异常
  • CallerRunsPolicy：当任务添加到线程池中被拒绝时，会使用调用线程池的Thread线程对象处理被拒绝的任务
  • DiscardOldestPolicy：当任务添加到线程池中被拒绝时，线程池会放弃等待队列中最旧的未处理任务，然后将被拒绝的任务添加到等待队列中
  • DiscardPolicy：当任务添加到线程池中被拒绝时，线程池将丢弃被拒绝的任务

ScheduledExecutorService

  • 将定时任务与线程池功能结合使用

Fork-join分治框架

并发集合框架

  • 7个非阻塞队列
    • ConcurrentHashMap 不支持排序
    • ConsurrentSkipListMap 支持排序
    • ConsurrentSkipListSet 支持排序而且不允许重复元素
    • ConcurrentLinkedQueue 并发队列操作 ，仅支持对列头操作
    • ConcurrentLinkedDeque 列头列尾双向操作
    • CopyOnWriteArrayList
    • CopyOnWriteArraySet
  • 阻塞队列
    • ArrayBlockingQueue
    • PriorityBlockingQueue
    • LinkedBlockingQueue
    • LinkedBlockingDeque
    • SynchronousQueue
    • DelayQueue
    • LinkedTransferQueue
